## ¿Qué es NOUS ACADEMY?

NOUS ACADEMY es una aplicación de escritorio Java diseñada para que academias privadas gestionen de forma centralizada todos sus procesos operativos. Desde la matrícula de estudiantes hasta la contratación de docentes, pasando por la programación de clases y el seguimiento financiero, NOUS ACADEMY cubre el ciclo completo de administración académica.

## Arquitectura del sistema

El software sigue un patrón MVC (Modelo-Vista-Controlador) con tres capas bien definidas:

- **Modelo** — Entidades de negocio: `Clase`, `Reserva`, `Grupo`, `Membresia`, `Contrato`
- **Controlador** — `AcademiaController`: punto único de acceso a toda la lógica de negocio
- **Vista** — Interfaz gráfica Swing con paneles específicos por rol de usuario

```
src/com/nousacademy/
├── boot/           # Carga inicial de datos persistidos
├── contabilidad/   # Membresías y contratos
├── controller/     # AcademiaController (lógica central)
├── excepciones/    # Excepciones de negocio tipadas
├── interfacegrafica/ # Paneles Swing por rol
├── modelo/         # Clases, reservas, grupos
├── pruebas/        # Pruebas de integración
└── usuarios/       # Usuarios, docentes, estudiantes
```

## Persistencia de datos

NOUS ACADEMY utiliza serialización de objetos Java para mantener el estado entre sesiones. Cada entidad se persiste en un archivo dedicado dentro del directorio `datos/`:

| Extensión | Entidad      |
|-----------|-------------|
| `.usr`    | Administrador |
| `.est`    | Estudiante   |
| `.doce`   | Docente      |
| `.res`    | Reserva      |
| `.grp`    | Grupo        |
| `.txt`    | Datos de la academia |

<Note>
  Al iniciar la aplicación, la clase `BootClass` deserializa automáticamente todos los archivos del directorio `datos/` y reconstruye el estado completo del sistema.
</Note>
