## Overview

`AcademiaController` is the single entry point for all business operations in NOUS ACADEMY. It holds the academy's internal state (collections of users, students, teachers, reservations, and groups) and delegates validation to the respective model constructors. Every mutating operation that touches persistent state also writes the affected file to `datos/`.

```java
package com.nousacademy.controller;

public class AcademiaController { ... }
```

<Note>
All arrays inside the controller start empty (`new X[0]`) and grow with `Arrays.copyOf`. There is no fixed capacity.
</Note>

## Constructor

```java
public AcademiaController(
    String nit,
    String nombre,
    String telefono,
    String correo,
    double capital
)
```

Initialises the academy with its identifying information and opening capital. All internal collections (`usuarios`, `estudiantes`, `docentes`, `reservas`, `grupos`) are created as empty arrays.

<ParamField path="nit" type="String" required>
  Tax identification number of the academy.
</ParamField>

<ParamField path="nombre" type="String" required>
  Legal name of the academy.
</ParamField>

<ParamField path="telefono" type="String" required>
  Contact phone number. Must be parseable as a number and non-negative.
</ParamField>

<ParamField path="correo" type="String" required>
  Academy email address. Must contain `@` and end with `nousacademy.co`.
</ParamField>

<ParamField path="capital" type="double" required>
  Opening capital of the academy in COP.
</ParamField>
